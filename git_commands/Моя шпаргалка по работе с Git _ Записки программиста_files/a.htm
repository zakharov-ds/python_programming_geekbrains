            Неточности.
            git add
            это команда добавления изменений файла в индекс.
            git commit
            это коммит того, что есть в индексе в локальный репозиторий.
            git commit -a
            автоматически добавляет в индекс измененные и удаленные файлы и коммитит их.
            Пишете:
            "Начать работать с веткой some_branch (уже существующей):
            git checkout -b some_branch origin/some_branch"
            На самом деле, команда создаёт новую ветку и сразу же переходит на неё.
            Археология, конечно, но шпаргалка не очень точная.
